# Definition of Done

## DoD documental

- Arquivos criados ou atualizados no local correto.
- Conteudo coerente com a visao do produto.
- Decisoes importantes registradas.
- Links principais revisados.
- Sem prometer funcionalidades que ainda nao existem.

## DoD frontend

- Escopo da sprint respeitado.
- Componentes pequenos e reutilizaveis quando fizer sentido.
- UX desktop consistente.
- Acessibilidade considerada.
- Sem regra de negocio pesada na UI.
- Build e testes aplicaveis executados.

## DoD Tauri/Rust

- Commands com contratos claros.
- Caminhos e permissoes validados.
- Erros tratados de forma segura.
- Nenhum acesso desnecessario ao filesystem.
- Testes ou validacao manual quando aplicavel.

## DoD storage

- SQLite usado para dados estruturados.
- JSON local usado para preferencias.
- Filesystem local usado para arquivos grandes.
- Migrations planejadas.
- Backup considerado.
- Dados do usuario preservados.

## DoD release

- Build validado.
- Artefatos identificados.
- Changelog ou notas de release.
- Instrucoes de instalacao coerentes.
- Limitacoes documentadas.

## DoD seguranca/privacidade

- Sem envio de dados sem consentimento.
- Sem telemetria escondida.
- Sem login obrigatorio no MVP.
- Dados locais protegidos contra caminhos inseguros.
- Politicas atualizadas.

## DoD open source

- Licenca definida antes de contribuicoes publicas.
- Codigo de conduta respeitado.
- Politica de conteudo respeitada.
- Documentacao de contribuicao atualizada.
- Sem materiais protegidos enviados ao repositorio.

## Criterios gerais

- Escopo respeitado.
- Documentacao atualizada.
- Build/teste quando aplicavel.
- Sem dependencias desnecessarias.
- Sem codigo morto.
- Sem features fora da sprint.
- Privacidade preservada.
- Dados do usuario protegidos.
- Conteudo e licencas respeitados.
